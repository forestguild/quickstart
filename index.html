Вебсайт гильдии

Эта страница поможет тебе построить вебсайт своей гильдии, просто следуй инструкции.
Здесь будет рассмотрена базовая версия на основе GitLab

Подготовка

Для работы с нуля тебе потребуется разобраться с несколькими инструментами и сервисами.
Инструменты

Git - инструмент управления исходным кодом, именно через него ты будешь управлять сайтом. Не беспокойся, все нужные команды будут тут же в инструкции, тебе не придется глубоко лезть.

Ruby - сайт использует движок на этом языке. По ссылке доступен установщик

Ruby Gems - если ты используешь Windows, открой cmd (Пуск - Выполнить - cmd.exe), в открывшемся окне напиши ridk install

Ruby Bundler - в той же консоли выполни gem install bundler

Сервисы

Для старта тебе нужен лишь GitLab, зарегистрируйся на нем.

Глава 1: запуск набора для быстрого старта

Здесь ты запустишь вот этот сайт (да, именно тот, что ты смотришь) у себя на компьютере, доступен он будет только тебе

Код

Для начала, нужно достать и подготовить исходный код вебсайта

  1. Открой консоль (в Windows - git bash или cmd), перейди в любую подходящую тебе папку
  2. Скопируй набор быстрого старта себе: git clone https://github.com/forestguild/quickstart
  3. Теперь у тебя есть все, что нужно для старта, перейди (в консоли) в директорую quickstart
  4. Установи все зависимости: bundle install (он может попросить тебя ввести пароль - все ок, не переживай)
  5. А теперь, магия: jekyll serve - эта команда соберет и запустит этот сайт, он станет доступен на http://127.0.0.1:4000
Настройка

А теперь, настройка самого сайта. Важный момент: при изменении _config.yml тебе нужно перезапустить команду jekyll serve

  1. Открой _config.yml (главный конфигурационный файл) и измени его под себя, там куча комментариев с пояснениями. Для примера, вот конфиг Ясного Леса
  2. Остальные настройки темы в папке _data/ui. К примеру, меню в шапке - _data/ui/navigation/top.yml, опять же - это можно сделать позже

Глава 2: welcome to the Internet

А здесь ты опубликуешь свой сайт в сети и сможешь поделиться адресом с гильдией :)

GitLab

Пришло время сохранить твои наработки в системе контроля версий (чтобы их не потерять)

  1. В папке с исходным кодом сайта удали папку .git (она может быть невидима)
  2. Создай новый проект на Gitlab, назови его (например) tirisfal и выбери видимость (Visibility Level) = Public
  3. В _config.yml установи в переменную baseurl название своего проекта, пример: baseurl: "/tirisfal" (нюансы Gitlab Pages)
  4. На открывшейся странице перейди к блоку кода "Existing folder" (существующая директория) и выполни инструкции оттуда
  5. Переходим к последнему пункту
Gitlab CI & Gitlab Pages

А теперь - публикация

  • Так как в исходном коде уже есть конфигурация для Gitlab CI (файл .gitlab-ci.yml), то тебе остается лишь перейти в Gitlab на страницу своего проекта → CI/CD → Pipelines. Тут будут отображаться процесс публикации твоего сайта
  • Адрес своего сайта ты найдешь на странице проекта в GitLab → Settings → Pages. В блоке "Access pages" ты найдешь адрес сайта
  • Вот и все :)

Глава 3: как жить дальше?

Сайт вроде-как сделал и даже опубликовал, а что дальше-то делать?

А вот тут начинается веселье, вот тебе список литературы на дом:

  • Jekyll - движок сайта, в его документации ты узнаешь как и что тут делается (начиная от добавления новых страниц и заканчивая разработкой "подкапотных" плагинов, которые будут делать "вжжжух!" магию
  • Typora - классный markdown редактор, который сэкономит тебе много нервов
  • GitHub Ясного Леса - вот тут лежит все, что используется в твоем сайте, включая документацию и примеры. Очень советую, если не понимаешь, как и нафига
Хочу крутых штук
  • Свой собственный домен сайта, к примеру: tirisfal.gg? Дальше сам: Документация GitLab
  • Автоматическое обновление сайта, чтобы прогресс гильдии обновлять, аттенданс трекер и т.д.? (например, раз в час)? Дальше сам: Документация GitLab
  • HTTPS, кэширование, быстрая работа? Дальше сам: CloudFlare
